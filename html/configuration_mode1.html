Configuration options

Frequency is the time between each sample taken by the temperature sensor. The default frequency is 10 minutes.

The number of samples is the amount of samples in the data files stored in the device. The web page only displays the last file created. The default number of samples si 120.

Samples transmision configuration

Information needed for set up the scp conection. This conection will send periodically the data stored in the device to a tarjet remote machine. This information is not stored in the device for security reasons, if the device shutdowns this configurations will be lost.

If you want to use this functionallity using also the power saving mode 3 you must accept to store this data the device checking the checkbox below else the scp will be desactivated when the device turn to power mode 3.

Power saving Modes

Normal Mode

In this mode the system works normally without any special configuration. The wifi module and the http server is active all the time.

Saving Mode 1 (Mode in use)

In this mode the system decreases the cpu performance. This mode reduces the heat generated by the cpu and increases the battery life time a bit. The wifi module and the http server will be active all the time.

Saving Mode 2

In this mode the system decreases the cpu performance and only switch on the wifi module and the http server in the times set by the user in the timetable. This mode increases de battery life time considerably.
